# Coding Standards

* Use camel case for all code and file identifiers where possible, except where in conflict with PSR or other naming standards.
* Use underscores and all lower case for all public-facing web assets and code files and executible code files.
* Use initial caps for all namespaces and class names.
* Use plural or category name for all database tables.
* Use singular category name for all namespaces.
* Use singular name for all file directories.
