# Declarative programming

There are two main styles of programming:

* Declarative programming, where you define a data structure that describes features or actions
* Imperative programming, where you describe a step-by-step process that accomplishes a task as a series of commands or functions

Declarative programming is easier to analyze statically. So using more declarative programming makes your program easy to understand.

The most basic example of this is the layered approach to web design. SQL, HTML, and CSS are all basically declarative in nature. They provide simple supporting layers around the imperative center of your program.

Other aspects of your program can be defined in a declarative way. For example, you can define your routes as a configuration file of routes. That is not only declarative but also centralized. The alternative is to implement all of your routes as special instructions scattered around your program, which is imperative and decentralized. In the first case, it is much easier to describe and understand your routes so it is a better design.
